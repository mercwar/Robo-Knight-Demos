## Development Guidelines

1. **AIFVS-ARTIFACT Headers**  
   Include the header in all demo files, shaders, and metadata objects.

2. **Visual Consistency**  
   Robo-Knight demos must preserve the canonical silhouette, armor geometry,
   and motion law.

3. **DX / Rendering Integrity**  
   Any rendering changes must be tested for stability and reproducibility.

## How to Submit

- Fork the repository  
- Create a feature branch (`git checkout -b feature/RK-Demo`)  
- Submit a Pull Request describing the visual or technical enhancement  

*Robo-Knight Law: Demos must shine with clarity and intent.*
